SPA(Single Page Application)
- 하나의 페이지에서도 데이터 변화를 보여주고싶어서 나옴(앱의 특성)
- 웹사이트를 앱으로 사용
- 하나의 페이지에서 어플리케이션처럼 돌길 바람
- 메모리에서 먼저 처리하고 화면을 돌림으로 페이지를 이동하지않아도 화면전환가능
- 모두 자바스크립트로 구성되어있음

react
- 무조건 노드 설치해야함
- 시작점은 index.js임
- 시작점이면서 끝점이 될 수 있기 때문에 전반적인 import를 다 넣어주면 전체적으로 import 반영

jsx와 js둘다 가능 
jsx에서 return은 자바스크립트 문법을 따음, 다만 화면을 구성하기위해 html과 유사한 기능들 이름들 제공

에러가 난다면 현재 인코드에 맞춰서 모듈들 버전 조절 npm update -> npm audit fix --force

-----------------------------리액트 공부----------------------------
1. 리액트는 spa임으로 하나의 페이지에서 나타남으로 html파일은 하나뿐임
    여기서도 public폴더에 index.html뿐임
2. index.js는 app.jsx와 index.html을 연결해줌 페이지가 변하는건 자바스크립트가 하는 일인데 얘가 연결해줌
3. index.js 설명
    <App>가 포인트 -> App를 데리고와서 root에 입힌다는 뜻
4. state 설명
- 변수의 값이 바뀔때마다 ui매번 업데이트해주기 어렵기때문에 state가 바뀔때마다 화면 ui업데이트 해주기로함

5. usestate사용법
[초기값저장된변수,업데이트함수] = useState(초기값)
업데이트함수(바뀐값)으로 state변형(비동기)
다만 초기값저장된 변수는 한박자씩 늦음 -> ui업데이트는 비동기적이라 바로 실행이 되는게 아니라 함수가 끝나면 처리됨으로 console.log하면 아직 업데이트되기 전의 값이 찍힘
또한, 업데이트함수를 보면 그게 들어있는 function은 무조건 실행됨으로 일반 변수는 업데이트할때 초기화된다 근데 그와중에 state는 저장하고있어서 초기화안됨
